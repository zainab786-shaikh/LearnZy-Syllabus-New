5.1 Basic Memory Management

What is Memory Management?

Memory management is how a computer organizes and controls its memory (RAM) to store and retrieve data efficiently. Think of it like organizing shelves in a library—if books (data) are stored properly, you can find and use them quickly.

When we run a program, it needs memory to execute. The Operating System (OS) allocates memory to different programs based on availability and requirements.

1. Partitioning in Memory Management

(a) Fixed Partitioning (Static Partitioning)

In fixed partitioning, memory is divided into fixed-sized blocks before running any program. Each program gets a block, but it may not use the entire space.

  • If a program is smaller than the partition, the remaining memory is wasted (internal fragmentation).
  • If a program is larger than the partition, it cannot be loaded (size limitation).

Example: Imagine a cinema hall with fixed-size seats (partitions). If a person (program) is smaller than the seat, extra space is wasted. If a person is bigger than the seat, they cannot fit in.

(b) Variable Partitioning (Dynamic Partitioning)

In variable partitioning, memory is divided into partitions dynamically as needed. No fixed sizes; partitions are created based on the program's size.

  • Eliminates internal fragmentation but can cause external fragmentation (small free spaces between partitions).

Example: Imagine a parking lot where cars (programs) park based on available space. A small car may leave some extra space, but at least it fits.

2. Free Space Management Techniques

(a) Bitmap Method

The memory is represented as a sequence of bits (0s and 1s).

  • 1 → Occupied block
  • 0 → Free block

The OS scans the bitmap to find free blocks and allocates them.

(b) Linked List Method

Memory blocks are organized as a linked list. Each block has a pointer to the next block.

  • Free List: Stores free memory blocks.
  • Allocated List: Stores occupied memory blocks.

Comparison Table: Fixed vs Variable Partitioning

Feature Fixed Partitioning Variable Partitioning
Partition Size Predefined (Fixed) Dynamic (Changes as needed)
Flexibility Low High
Internal Fragmentation High No
External Fragmentation No Yes
Efficient Memory Usage Low High

Final Thoughts

  • Fixed Partitioning is simple but wastes memory.
  • Variable Partitioning is flexible but can cause fragmentation.
  • Bitmap is an easy way to track free memory but is slow.
  • Linked List is more efficient but needs extra storage.

Memory management is crucial for computers to run multiple programs efficiently. The OS decides which technique to use based on system requirements.
