3.2 Process Scheduling

Process scheduling is the method by which the operating system (OS) manages the execution of multiple processes in a computer. Since the CPU can only execute one process at a time (in a single-core system), the OS ensures that all processes get a fair chance to execute by scheduling them efficiently.

To understand Process Scheduling, we will break it down into the following parts:

  • Scheduling Queues
  • Schedulers
  • Context Switch

1. Scheduling Queues

A queue is like a waiting line where processes stand in order to get executed by the CPU. In an operating system, there are different types of queues to manage processes.

Types of Scheduling Queues

Job Queue

This queue contains all processes that are ready to be executed.

Example: When you open multiple applications on your computer (such as a web browser, music player, and Word document), all these processes enter the job queue.

Ready Queue

This queue contains all processes that are ready to run and are waiting for CPU time.

Example: Suppose you have 5 apps running, but the CPU can only process one at a time. The remaining 4 processes wait in the ready queue.

Waiting Queue (I/O Queue)

If a process needs to wait for some input/output (I/O) operation (e.g., reading a file, printing a document), it moves to the waiting queue.

Example: When you click Print in MS Word, the document is placed in a queue until the printer is available.

Example of Scheduling Queues in Real Life

Imagine you are in a restaurant:

  • Job Queue → All customers who enter the restaurant and are waiting for a table.
  • Ready Queue → Customers who got a table and are waiting for their food.
  • Waiting Queue → Customers who ordered food and are waiting for the chef to prepare it.

2. Schedulers

Schedulers are responsible for managing processes and moving them between different queues.

Types of Schedulers

1. Long-Term Scheduler (Job Scheduler)

Decides which processes should enter the ready queue from the job queue.

Example: Imagine you are at an airport. Job Scheduler decides which passengers should go through security checks (entering the Ready Queue).

2. Short-Term Scheduler (CPU Scheduler)

Decides which process gets CPU time next from the ready queue.

Example: A teacher asks students one by one to answer a question. The teacher decides who answers next (similar to how the short-term scheduler picks the next process).

3. Medium-Term Scheduler

Removes processes from memory (called swapping out) and brings them back later (called swapping in).

Example: Imagine a restaurant with limited seats. If it’s too crowded, the manager may ask some people to wait outside until seats are available. This is similar to swapping out processes to free up memory.

3. Context Switch

A context switch happens when the CPU switches from one process to another. It saves the state of the current process and loads the state of the new process.

How Context Switching Works

  • Save the current process’s state (like remembering what a person was doing before taking a break).
  • Load the next process’s state (like picking up where another person left off).
  • Start executing the new process.

Example of Context Switching in Real Life

Imagine a teacher calls on Student A to answer a question. Suddenly, the teacher switches to Student B without letting Student A finish. The teacher must remember where Student A stopped before returning to them.

Final Summary

  • Scheduling Queues: Help manage processes waiting to be executed.
  • Schedulers: Decide which processes get CPU time and when.
  • Context Switch: Saves and switches processes when multitasking.
