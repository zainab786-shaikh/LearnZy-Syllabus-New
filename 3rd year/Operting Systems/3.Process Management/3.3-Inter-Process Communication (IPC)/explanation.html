3.3 Inter-Process Communication (IPC)

Introduction to IPC

Inter-Process Communication (IPC) is a mechanism that allows multiple processes to communicate with each other. Processes are independent programs running on a computer, and sometimes they need to share information or coordinate tasks. IPC helps in achieving this by enabling data exchange between processes.

Why is IPC needed?

  • When multiple processes work on the same task, they may need to share data.
  • Some processes may need to send messages to others to coordinate their operations.
  • IPC helps in better resource utilization by allowing processes to work together efficiently.
  • It is essential in multi-core and distributed computing systems.

Examples of IPC in Daily Life

  • Chat Applications – When you send a message on WhatsApp, your message (process 1) is sent to the server (process 2), which then delivers it to your friend (process 3).
  • Google Docs – When multiple users edit a document, different processes share and update data simultaneously.
  • Banking Systems – When you withdraw money from an ATM, the bank server process communicates with the ATM process to verify your balance and process the transaction.

Shared Memory System

In a shared memory system, multiple processes communicate by reading and writing data into a common memory space. This is like a shared whiteboard where multiple people can write and read information.

How Shared Memory Works?

  • One process creates a shared memory segment.
  • Other processes attach to the shared memory segment.
  • Processes read from and write to the shared memory to exchange information.
  • Once communication is done, processes detach from the shared memory.

Example of Shared Memory

Imagine two employees in an office using a whiteboard to exchange messages:

  • Employee A writes “Meeting at 3 PM” on the whiteboard.
  • Employee B reads it and notes the time. This is similar to how shared memory works in IPC.

Advantages of Shared Memory

  • Fast Communication – Since no extra data copying is required.
  • Efficient for large data – Useful when processes need to share a lot of information quickly.

Disadvantages of Shared Memory

  • Synchronization issues – If multiple processes modify data at the same time, conflicts may occur.
  • Security risks – One process can accidentally overwrite important data.

Message Passing System

In a message passing system, processes communicate by sending and receiving messages. Unlike shared memory, processes do not directly share data but instead exchange messages through predefined communication mechanisms.

How Message Passing Works?

  • One process sends a message to another process.
  • The receiving process gets the message and processes it.
  • The system ensures that messages are delivered in the correct order.

Example of Message Passing

Imagine two people talking over a walkie-talkie:

  • Person A presses a button and says, “Let’s meet at 3 PM.”
  • Person B listens and responds, “Okay, see you then.” Here, the voice messages are like IPC message passing.

Types of Message Passing

  • Synchronous Message Passing – The sender waits for the receiver to acknowledge the message before continuing.
    Example: A phone call where both people must be present to communicate.
  • Asynchronous Message Passing – The sender sends the message and continues its work without waiting for the receiver.
    Example: Sending an email where the receiver can read it later.

Advantages of Message Passing

  • No direct memory sharing – This makes it safer and avoids data conflicts.
  • Better security – Each message is controlled and structured properly.

Disadvantages of Message Passing

  • Slower than shared memory – Messages need to be copied and processed.
  • Overhead in communication – Extra steps are needed to ensure message delivery and order.

Comparison of Shared Memory and Message Passing

Feature Shared Memory Message Passing
Speed Faster Slower
Complexity More complex (needs synchronization) Simpler to implement
Security Less secure More secure
Data Transfer Direct memory access Needs message transmission
Best for Large data exchange Simple and structured communication

Conclusion

Inter-Process Communication (IPC) is essential for processes to collaborate effectively. Shared memory provides fast communication but requires synchronization, whereas message passing is structured and secure but can be slower. Choosing the right IPC mechanism depends on the application’s requirements.
