6.2 Access Methods & File Allocation Methods

Access Methods

Access methods define how data is read from or written to a storage device like a hard disk, SSD, or even a USB drive. The main types of access methods are:

1. Sequential Access

In sequential access, data is read or written in a specific order, one after another, just like a tape recorder.

Example: Imagine you have a music playlist on an old cassette tape. If you want to listen to song number 5, you cannot directly jump to it. You have to play (or fast-forward) through songs 1, 2, 3, and 4 first.

Real-World Example:

  • Magnetic tapes used for backups work with sequential access.
  • Movie streaming: When you watch a movie online, the video plays from start to end in sequence.

2. Direct Access (Random Access)

Direct access allows you to jump directly to the required data without reading everything before it.

Example: Imagine a book where each chapter has a bookmark. If you need to read Chapter 5, you directly flip to that page instead of reading from Chapter 1.

Real-World Example:

  • Hard drives (HDDs) and SSDs use direct access to retrieve files quickly.
  • USB flash drives allow you to open any file directly without going through others.

3. Swapping

Swapping is a memory management technique where processes (running programs) are moved between RAM (main memory) and disk (storage) to free up space.

Example: Think of a classroom where students take turns sitting at a desk. If there aren’t enough desks, some students wait outside (in storage). When a student inside is done, another one takes their place.

Real-World Example: In computers, if you open too many applications and RAM gets full, the operating system temporarily moves some inactive applications to disk storage (this is called the swap file or virtual memory).

File Allocation Methods

File allocation methods define how files are stored on a disk. The three main types are:

1. Contiguous Allocation

In contiguous allocation, a file is stored in one continuous block of memory.

Example: Imagine books in a library arranged in consecutive shelves. If you need a book, all its pages are together, making it easy to read.

Advantages:

  • Fast access speed (because data is stored in one place).
  • Easy to implement.

Disadvantages:

  • Wasted space: If a file is smaller than the allocated space, the extra space is wasted.
  • Fragmentation: If a large file cannot fit into a single block, it becomes hard to store.

Real-World Example:

  • Old floppy disks used contiguous allocation.
  • Some database systems store data in contiguous blocks for faster retrieval.

2. Linked Allocation

In linked allocation, a file is stored in different scattered blocks, and each block contains a pointer to the next block.

Example: Think of a treasure hunt where each clue (block) leads you to the next location (next block).

Advantages:

  • No fragmentation issues since blocks don’t have to be continuous.
  • Files can expand dynamically.

Disadvantages:

  • Slower access speed because the system must follow links to find data.
  • If a pointer is lost or damaged, the entire file can become unreadable.

Real-World Example:

  • USB drives use linked allocation to store fragmented files.
  • Operating systems like Linux and Windows sometimes use linked allocation for storing log files.

3. Indexed Allocation

In indexed allocation, a separate index table stores all the addresses of a file’s blocks.

Example: Think of a recipe book’s index page where each dish’s name points to its page number. You don’t need to flip through all pages; just check the index and go to the right page.

Advantages:

  • Direct access to any part of the file.
  • No fragmentation problem.

Disadvantages:

  • Requires extra space for storing the index table.
  • If the index table is lost, all data may become inaccessible.

Real-World Example:

  • Hard disks and SSDs use indexed allocation for storing operating system files.
  • File systems like NTFS (Windows) and ext4 (Linux) use indexed allocation for faster data retrieval.

Comparison of File Allocation Methods

Method Storage Type Speed Space Efficiency Example
Contiguous Continuous blocks Fast Wastes space Floppy disks, databases
Linked Scattered blocks Slow Efficient USB drives, log files
Indexed Index table-based Medium Needs extra space for index NTFS, ext4 file system

Conclusion

Understanding Access Methods and File Allocation Methods is crucial for managing data efficiently in storage systems.
