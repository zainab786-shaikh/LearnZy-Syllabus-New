6.3 Directory Structure and Disk Organization

Every computer system needs a way to store and manage files efficiently. The directory structure organizes these files, while the disk organization determines how they are physically and logically stored.

I. DIRECTORY STRUCTURE

1. Single-Level Directory

This is the simplest type of directory structure.

  • All files are stored in a single directory.
  • No subdirectories (folders inside folders).

Example: Imagine a box where you keep all your documents together without any dividers.

Advantages:

  • Easy to manage.
  • Simple file access.

Disadvantages:

  • Name conflicts.
  • Difficult to organize files.

2. Two-Level Directory

There is a separate directory for each user.

Example: Each student has their own locker.

Advantages:

  • Users have private directories.
  • No filename conflicts.

Disadvantages:

  • Users cannot share files easily.

3. Tree-Structured Directory

A hierarchical structure where directories can have subdirectories.

Example: A computer‚Äôs C: drive structure.

Advantages:

  • Better organization.
  • Multiple levels of folders.

Disadvantages:

  • More complex than a single-level directory.
  • Searching for files may take more time.

II. DISK ORGANIZATION AND STRUCTURE

1. Physical Structure of Disk

A hard disk consists of circular plates called platters.

Example: A vinyl record (old music disc).

Components of a Disk:

  • Platters
  • Tracks
  • Sectors
  • Read/Write Head

2. Logical Structure of Disk

Determines how data is stored and retrieved.

Example: A library index.

Key Logical Structures:

  • Boot Block
  • Superblock
  • File Control Block (FCB)
  • Data Blocks

3. RAID Structure of Disk

RAID (Redundant Array of Independent Disks) combines multiple hard disks to improve performance and reliability.

Why use RAID?

  • Speed
  • Redundancy
  • Large Storage

III. RAID LEVELS (0 to 6)

RAID 0 ‚Äì Striping

Data is split across multiple disks.

Example: Dividing a book into separate chapters.

Pros: Very fast.

Cons: No data protection.

RAID 1 ‚Äì Mirroring

Data is copied identically on two disks.

Example: Keeping two identical copies of a document.

Pros: Full data protection.

Cons: Storage space is wasted.

RAID 5 ‚Äì Block-Level Striping with Distributed Parity

Data and parity information are distributed across all disks.

Example: Making photocopies of different sections of a document.

Pros: Good speed + data protection.

Cons: More complex.

RAID 6 ‚Äì Double Parity

Like RAID 5 but with extra parity for more security.

Example: Extra backup copy.

Pros: Highly reliable.

Cons: Slower write speed.

Conclusion

  • Directory structures help in organizing files.
  • Disk organization determines how data is arranged.
  • RAID levels provide different methods for storage speed and data protection.
