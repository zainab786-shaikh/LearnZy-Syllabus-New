6.3 Exception Handling in Python

Introduction to Exception Handling

While writing programs, errors are inevitable. Some errors occur due to mistakes in the code (syntax errors), while others occur when the program runs (runtime errors). These runtime errors are called exceptions.

For example, dividing a number by zero or trying to open a file that does not exist will cause exceptions. Instead of letting the program crash, we can handle such errors using exception handling in Python.

What Happens When an Exception Occurs?

If an exception occurs and is not handled, the program stops immediately and displays an error message.

      
        a = 5
        b = 0
        print(a / b)  # ZeroDivisionError

Exception Handling Using try-except Statement

Python provides the try-except block to catch and handle exceptions gracefully.

      
        try:
            risky_code()
        except:
            handle_error()

Example 1: Handling Division by Zero

      
        try:
            a = 5
            b = 0
            print(a / b)  # This will cause an error
        except:
            print("Oops! Division by zero is not allowed.")

Catching Specific Exceptions

We can handle different types of exceptions separately.

      
        try:
            num = int(input("Enter a number: "))
            print(10 / num)
        except ZeroDivisionError:
            print("You cannot divide by zero!")
        except ValueError:
            print("Invalid input! Please enter a number.")

Using the raise Statement

Sometimes, we need to intentionally raise an exception using the raise keyword.

      
        age = int(input("Enter your age: "))
        if age < 18:
            raise ValueError("Sorry, you must be 18 or older.")
        else:
            print("You are allowed to continue.")

User-Defined Exceptions

      
        class AgeTooSmallError(Exception):
            pass
        
        try:
            age = int(input("Enter your age: "))
            if age < 18:
                raise AgeTooSmallError
            print("Access granted!")
        except AgeTooSmallError:
            print("You are too young!")

Using finally to Run Cleanup Code

      
        try:
            f = open("example.txt", "r")
            print(f.read())
        except FileNotFoundError:
            print("File not found!")
        finally:
            print("Cleaning up resources...")

Conclusion

  • Exceptions are runtime errors that can crash a program if not handled.
  • We use try-except to catch and handle exceptions.
  • We can handle multiple exceptions separately.
  • The raise statement allows us to generate custom exceptions.
  • We can create user-defined exceptions for specific cases.
  • The finally block ensures cleanup operations always run.
