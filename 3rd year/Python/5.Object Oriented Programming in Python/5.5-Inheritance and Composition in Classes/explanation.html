5.5 Inheritance and Composition in Classes

In object-oriented programming (OOP), Inheritance and Composition are two important ways to create relationships between classes. These concepts help in code reusability, modularity, and organization of programs.

1. Inheritance (Is-a Relationship)

What is Inheritance?

Inheritance is a mechanism that allows a new class to inherit properties and methods from an existing class. This helps in avoiding repetition and makes the code more organized and reusable.

  • Parent Class (Super Class) – The class whose properties and methods are inherited.
  • Child Class (Sub Class) – The class that inherits from the parent class.

Example of Inheritance:

Imagine you are designing a system for different types of vehicles. Instead of writing the same attributes (like speed, color, brand) for every type of vehicle, we can create a base class and let other vehicle types inherit from it.

      
  # Parent class
  class Vehicle:
      def __init__(self, brand, color, speed):
          self.brand = brand
          self.color = color
          self.speed = speed
  
      def show_info(self):
          print(f"Brand: {self.brand}, Color: {self.color}, Speed: {self.speed} km/h")
  
  # Child class (inherits from Vehicle)
  class Car(Vehicle):
      def __init__(self, brand, color, speed, fuel_type):
          super().__init__(brand, color, speed)
          self.fuel_type = fuel_type
  
      def show_car_info(self):
          self.show_info()
          print(f"Fuel Type: {self.fuel_type}")

2. Composition (Has-a Relationship)

What is Composition?

Composition is when a class contains an object of another class as a part of its attributes. Instead of inheriting, it uses another class inside it.

Example of Composition:

      
  # Separate class for Engine
  class Engine:
      def __init__(self, horsepower, type):
          self.horsepower = horsepower
          self.type = type
  
      def show_engine_info(self):
          print(f"Engine: {self.horsepower} HP, Type: {self.type}")
  
  # Car class has an Engine (Composition)
  class Car:
      def __init__(self, brand, color, speed, engine):
          self.brand = brand
          self.color = color
          self.speed = speed
          self.engine = engine
  
      def show_car_info(self):
          print(f"Brand: {self.brand}, Color: {self.color}, Speed: {self.speed} km/h")
          self.engine.show_engine_info()

3. Key Differences Between Inheritance and Composition

Feature Inheritance (IS-A) Composition (HAS-A)
Definition One class derives from another One class contains an object of another
Example Car is a Vehicle Car has an Engine
Relationship Strong relationship (Parent-Child) Loose relationship (Objects work together)

4. When to Use Inheritance vs Composition?

Situation Use Inheritance Use Composition
Objects are logically related (e.g., "A Car is a Vehicle") ✅ Yes ❌ No
One class contains another class (e.g., "A Car has an Engine") ❌ No ✅ Yes

5. Real-Life Examples of Inheritance & Composition

Scenario Inheritance Composition
School System Teacher inherits from Person Teacher has a Subject
Game Development Enemy inherits from Character Game has a Scoreboard

Conclusion

Use Inheritance when one class "is a" type of another.

Use Composition when one class "has a" relationship with another.

Both Inheritance and Composition are useful depending on the situation. The key is to choose the right approach based on the relationship between the objects.
